# TD-001: 路径逃逸防护薄弱

## 元信息
| 字段 | 值 |
|------|-----|
| ID | TD-001 |
| 分类 | 安全风险 / OWASP漏洞 |
| 严重程度 | critical |
| 状态 | Open |
| 发现日期 | 2026-06-26 |
| 解决日期 | - |

## 描述
`GenService.genCodeData` 中对 `projectBasePath` 的路径逃逸防护采用字符串替换：

```java
projectBasePath = projectBasePath.replace("../", "parent/");
```

该实现仅能拦截字面量 `../`，无法应对以下绕过手段：
- `..\\`（Windows 反斜杠）
- `..%2F`、`..%5C`（URL 编码）
- `....//`（替换后仍残留 `../`）
- 多层嵌套编码

由于 `projectBasePath` 来自用户可配置的 `gen_task.project_base_path`，攻击者可构造恶意路径使生成文件写到 `distPath` 之外的任意位置。

## 影响范围
- 影响所有调用 `getGenZip` / `genCodeData` 的生成请求。
- 可能导致任意文件写入，覆盖服务器关键文件。

## 复现条件
1. 创建一个生成任务，将 `projectBasePath` 设为 `../../../etc` 或 `..\\..\\windows\\system32`。
2. 触发代码生成。
3. 观察生成文件是否落到 `distPath` 之外。

## 当前解决方案（如有）
仅做 `../` 字面量替换，无规范化校验。

## 建议解决方案
- 使用 `Path.normalize()` + `Path.startsWith(distPath)` 做严格的路径包含校验，渲染前断言最终文件路径必须位于 `distPath` 之下。
- 拒绝任何包含 `..` 段的 `projectBasePath` 输入。
- 在 Rest 层对 `projectBasePath` 做白名单字符校验（仅允许字母、数字、`/`、`-`、`_`）。
