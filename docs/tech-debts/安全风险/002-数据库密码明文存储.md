# TD-002: 数据库密码明文存储

## 元信息
| 字段 | 值 |
|------|-----|
| ID | TD-002 |
| 分类 | 安全风险 / 数据泄露 |
| 严重程度 | high |
| 状态 | Open |
| 发现日期 | 2026-06-26 |
| 解决日期 | - |

## 描述
`GenDatasource.dbPassword` 以明文形式存储于 `gen_datasource` 表。虽然 `GenDatasourceRest.detail` 接口在返回前将 `dbPassword` 置为 `null` 实现了前端脱敏，但：

1. 数据库层面密码仍为明文，一旦数据库被拖库或备份泄露，全部目标业务库密码直接暴露。
2. `page` 接口与其他未显式脱敏的查询路径可能将明文密码带出。
3. 日志中若打印 `GenDatasource` 实体（如 `log.info`），密码会进入日志。

## 影响范围
- `gen_datasource` 表全部记录。
- 所有目标业务库的连接凭据。

## 复现条件
1. 创建一个数据源，填入真实密码。
2. 直接查询 `gen_datasource` 表，可见明文密码。

## 当前解决方案（如有）
仅在 `detail` 接口返回前置 `dbPassword=null`，存储层未做加密。

## 建议解决方案
- 存储层对 `dbPassword` 做对称加密（如 AES），密钥由配置中心或环境变量管理。
- 在 MyBatis TypeHandler 中统一加解密，避免业务层感知。
- 全部查询路径统一脱敏，或在实体序列化时用 `@JsonIgnore` / `@Sensitive` 注解控制。
- 审计日志中禁止打印 `GenDatasource` 实体。
