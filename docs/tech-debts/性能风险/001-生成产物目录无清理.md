# TD-004: 生成产物目录无清理机制

## 描述
`GenService.getDistPath` 每次生成在 `{catalina.base}/gen/{yyyyMMddHHmmssSSS}/` 下新建目录，渲染产物与 zip 全部落盘：

```java
private static String getDistPath() {
    String property = System.getProperty("catalina.base");
    String genPath = property + "/gen/" + dateStr;
    ...
}
```

`zipDataAndPush` 在 distPath 同级生成 `.zip` 文件后，既未删除原始 distPath 目录，也未删除 zip 文件。长期运行下：
1. 磁盘空间持续增长，最终可能打满。
2. 大量小文件拖慢文件系统索引。

## 影响范围
- 服务器 `{catalina.base}/gen/` 目录。
- 长期运行的生产/测试环境。

## 复现条件
1. 连续触发 N 次代码生成。
2. 检查 `{catalina.base}/gen/` 目录，可见 N 个时间戳子目录与 N 个 zip 文件，无任何清理。

## 当前解决方案（如有）
无自动清理，依赖人工或外部定时任务。

## 建议解决方案
- 生成完成后（zip 推送成功后）删除原始 distPath 目录与 zip 文件，改为流式打包避免落盘。
- 或引入保留策略（如仅保留最近 7 天产物），定时清理过期目录。
- 在 `gen_log` 中标记清理状态，便于追踪。

## 关联模块
- `generator-server/src/main/java/com/wkclz/generator/server/service/GenService.java`（getDistPath、zipDataAndPush 方法）
