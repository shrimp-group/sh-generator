## 核心规则详细描述

### 规则分类

业务规则可分为以下几类：

1. **校验规则** - 数据校验和约束（项目-数据源绑定、任务-项目一致性、任务-模板唯一性、空数据校验）
2. **计算规则** - 业务计算逻辑（产物时间戳目录）
3. **流转规则** - 状态流转条件（任务 createSwitch / deleteSwitch 控制生成行为）
4. **权限规则** - 访问控制策略（`userCode` 数据隔离、密码脱敏）
5. **安全规则** - 路径防逃逸、列过滤

### 规则文档结构

每个业务规则文档应包含：

1. **规则概述** - 规则目的和背景
2. **规则表达式** - 规则的正式定义
3. **触发时机** - 何时执行该规则
4. **执行结果** - 规则执行后的影响
5. **例外情况** - 规则的特殊情况
6. **变更历史** - 规则的修改记录

### 关键规则补充说明

#### 路径防逃逸

当前实现为 `projectBasePath.replace("../", "parent/")`，属于字符串替换而非严格的路径规范化。该规则能阻止最常见的 `../` 逃逸，但无法应对 `..\\`、`..%2F`、多层编码等绕过手段，存在改进空间（详见技术债务记录）。

#### 列过滤-deleted

`getColumns` 在读取列元数据后通过 `columns.stream().filter(t -> !"deleted".equals(t.getColumnName())).toList()` 硬编码过滤 `deleted` 列。该规则假定所有目标表都遵循统一的 `deleted` 软删除列命名约定。

### 规则命名规范

- 规则名采用有意义的描述性名称
- 规则ID采用小写下划线命名，如 `project_dbcode_binding_rule`
